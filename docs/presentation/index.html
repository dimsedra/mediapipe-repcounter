Langkah 1 — Rumusan Masalah

Mengapa Memilih Visi Komputer Dibandingkan Sensor Fisik?

Untuk menjawab kebutuhan tersebut, kita harus memilih teknologi pemantau yang paling praktis dan dapat dijangkau masyarakat luas.

Sensor Wearable / MoCap (Hambatan Biaya)
Sensor fisik IMU atau kamera MoCap laboratorium berbiaya mahal (jutaan rupiah), rumit dipakai, dan tidak praktis untuk latihan harian di rumah.
Visi Komputer Kamera RGB (Solusi Praktis)
Memanfaatkan webcam laptop atau HP biasa tanpa alat tambahan. Pengguna cukup berdiri di depan kamera, lalu sistem langsung menghitung.
→ Tantangan Baru: Kamera RGB memang praktis, tetapi bagaimana jika algoritmanya tidak tahan gangguan lingkungan?
Langkah 2 — Analisis Kondisi Saat Ini

Kegagalan Algoritma Visi Komputer Konvensional

Saat menguji algoritma konvensional berbasis ambang batas tunggal (single threshold), ditemukan masalah kritis yang memalsukan data pengguna.

Fenomena Signal Chatter (Hitungan Ganda):
Saat atlet menahan posisi atau sedikit bergetar di titik terendah squat, sinyal sudut tubuh melintasi garis ambang tunggal secara berulang. Akibatnya, 1 gerakan squat dihitung sebagai 2 hingga 4 kali.
→ Titik Tolak QCC: Berdasarkan kegagalan fatal ini, proyek QCC dirancang untuk menciptakan sistem yang 100% tahan chatter.

Langkah 3 — Akar Penyebab Utama

Dua Akar Penyebab Eror Terbesar

Dari analisis Fishbone, ditemukan dua faktor utama yang paling sering merusak keakuratan visi komputer:

1. Geometri Kamera Depan (0°)
Proyeksi 2D dari sudut depan mengalami perspective foreshortening. Gerakan fleksi lutut terkompresi tegak lurus terhadap lensa, sehingga kedalaman squat tampak jauh lebih dangkal dari aslinya.
2. Pakaian Longgar (Baggy Clothes)
Lipatan kain celana longgar di bawah pencahayaan redup menciptakan bayangan buatan yang menggeser lokasi titik sendi lutut asli MediaPipe hingga 15–20 pixel.
→ Solusi QCC: Bagaimana kita merancang algoritma baru untuk menjawab 2 akar masalah ini?
Langkah 4 — Rencana Solusi / Penanggulangan

3 Pilar Inovasi Solusi QCC

Untuk mengatasi chatter sinyal, variasi postur atlet, dan gangguan lingkungan, tim merancang 3 penanggulangan utama:

1. Sudut 2D Invariant
Menghitung sudut interior hip–knee–ankle 2D murni yang bebas dari pengaruh tinggi badan atlet maupun jarak kamera.
2. Auto-Kalibrasi Berdiri
Mengukur sudut berdiri tegak di awal sesi untuk menentukan ambang kedalaman dinamis secara spesifik per individu.
3. 2-Threshold Hysteresis
Menambahkan margin +5° pada transisi status untuk mengunci dan menghentikan hitungan ganda akibat getaran sinyal.
→ Implementasi: Mari kita lihat bagaimana 3 pilar ini dirangkai ke dalam arsitektur sistem.

Sisi Positif Penolakan FSM
FSM Hysteresis berhasil 100% menolak gerakan dangkal (0 rep sah dikreditkan dari 10 trial), mencegah memalsukan hitungan.
Akar Masalah Ambang Sudut Partial
Banyak partial squat terlalu dangkal (120°–135°) tidak terdeteksi statusnya karena ambang pemicu FSM terlalu ketat, dianggap sebagai standing sway.
→ Rekomendasi FSM: Perlu penyesuaian ambang pemicu entri (θentry = θstanding - 15°) untuk peringatan visual "Squat Kurang Dalam".

Langkah 7 — Standarisasi Operasional (SOP)

Pencegahan Masalah Berulang (SOP Penggunaan)

Berdasarkan seluruh temuan eksperimen, dirumuskan 4 aturan standar operasional untuk mencegah eror terulang:

1. Posisi Kamera Ideal
Wajib menempatkan kamera pada sudut Samping (90°) atau Serong (45°) jarak 1.8–2.0 m. Hindari posisi depan.
2. Interactive Setup Check
Sistem wajib memeriksa visibilitas landmark sendi (≥ 0.5) dan framing sebelum sesi dimulai.
3. Pakaian Olahraga Pas
Pengguna dianjurkan menggunakan pakaian pas (fitted) untuk menghindari oklusi kain pada sendi lutut.
4. Penggunaan Koordinat 2D
Gunakan 2D world coordinates murni dan abaikan kanal kedalaman z monokular yang tidak stabil.

Langkah 8 — Rencana Tindak Lanjut

Roadmap Pengembangan Masa Depan

Untuk memperluas dampak keberhasilan QCC ini, disusun 3 langkah pengembangan jangka panjang:

1. Form Feedback Biomekanika
Menambahkan deteksi koreksi postur (knee valgus, trunk lean angle, heel lift, dan tempo gerakan).
2. Multi-Person Tracking
Mengintegrasikan ID tracking (ByteTRACK) untuk mendukung sesi olahraga kelompok di studio fitnes / gym.
3. Aplikasi Mobile Native
Memporting logika FSM ke Android (Kotlin) & iOS (Swift) memanfaatkan API Tasks Vision yang fleksibel.
